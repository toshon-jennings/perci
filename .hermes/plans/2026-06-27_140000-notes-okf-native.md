# Plan: Notes Native OKF Storage

## Goal
Make Perci Notes save natively in Open Knowledge Format (OKF v0.1) with a
collapsible, always-visible metadata panel above the editor. Users should be
able to edit OKF fields without being forced to — auto-population handles
missing fields gracefully.

## OKF v0.1 Spec Summary (from GoogleCloudPlatform/knowledge-catalog)
- Markdown files with YAML frontmatter
- Required: `type`
- Recommended: `title`, `description`, `resource`, `tags`, `timestamp`
- Standard markdown cross-links (relative or `/` absolute)
- Optional `index.md` per directory, `log.md` for history
- Permissive: broken links and unknown types are fine

## Current State
- Notes already parses frontmatter via `src/lib/notesTags.js` (tags field only)
- Notes already has `[[wikilinks]]` and backlinks panel
- Notes already has "Export OKF" button (one-way conversion, not native storage)
- MonacoEditor is the editor component
- Preview uses `react-markdown` with `remarkGfm`
- Frontmatter is stripped before preview (`stripNoteFrontmatter`)

## Approach

### 1. Extend `src/lib/notesTags.js` → `src/lib/notesOKF.js`

Expand the existing frontmatter utilities to handle the full OKF schema:

- `parseNoteOKF(content)` → returns `{ type, title, description, resource, tags, timestamp, body }`
- `buildNoteOKF(fields, body)` → returns full markdown string with frontmatter
- `ensureOKFDefaults(fields, fileName)` → fills in auto-generated values:
  - `type`: default "Note"
  - `title`: derive from filename or first H1
  - `description`: derive from first non-empty body line (truncated to 100 chars)
  - `timestamp`: current ISO 8601 datetime
- `migrateLegacyFrontmatter(content)` — handles notes that have `tags:` but not full OKF

### 2. Add OKF Metadata Panel (above editor)

New component: `src/components/NotesOKFPanel.jsx`

Layout:
```
┌─────────────────────────────────────────────────┐
│ ▼ OKF Metadata   (collapsible header)           │
│   type: [Note    ]  title: [_______________]    │
│   description: [________________________]        │
│   resource: [____________________________]      │
│   tags: [architecture] [decision] [+add]        │
│   timestamp: 2026-06-27T14:30:00Z               │
└─────────────────────────────────────────────────┘
│                                                 │
│           MonacoEditor (existing)               │
│                                                 │
```

Behavior:
- Collapsed by default (shows summary line: "type: Note | 2 tags")
- Click to expand → editable fields
- All fields optional — empty fields use auto-derived values on save
- `type` dropdown with common values: Note, Decision, Concept, Reference, Playbook
- Tags editor reuses existing tag input pattern
- `resource` is a URL field
- `timestamp` is auto-set, read-only display

### 3. Modify Save/Load Flow

**On save (`saveNoteToDisk`):**
1. Parse current OKF from editor content (or existing file content)
2. Merge with panel field values
3. Ensure defaults via `ensureOKFDefaults`
4. Rebuild frontmatter + body → write to disk

**On load (`loadNotes` / `handleSelectNote`):**
1. Read file content
2. Parse OKF frontmatter
3. If no OKF frontmatter exists:
   - Check for legacy `tags:` frontmatter → migrate
   - If no frontmatter at all → generate OKF defaults from filename + body
4. Set panel state from parsed fields
5. Editor receives full content (frontmatter + body) — Monaco shows it

### 4. Preview Update

- Currently: `stripNoteFrontmatter(unsavedContent)` before rendering
- Change: Keep frontmatter visible in preview as a metadata card at top, or strip it and show OKF fields from panel state (not from parsed frontmatter, to avoid duplication with the panel)
- Decision: Strip from preview body (cleaner), OKF info is already in the panel above

### 5. Index Maintenance

- On each save, update `index.md` in the notes folder
- Format: one line per note with title + description + link
- Simple append/update logic (no need for full OKF index spec compliance)

### 6. Files to Change

| File | Change |
|------|--------|
| `src/lib/notesOKF.js` (new) | OKF parse/build/defaults/migration utilities |
| `src/components/NotesOKFPanel.jsx` (new) | Collapsible metadata panel component |
| `src/components/NotesMode.jsx` | Integrate panel, update save/load to use OKF, maintain index |
| `src/components/NotesMode.css` | Styles for OKF panel (collapsible header, fields) |

### 7. Backward Compatibility

- Existing notes without OKF frontmatter → auto-migrated on first open
- Notes with legacy `tags:` frontmatter → `migrateLegacyFrontmatter` preserves tags, adds other fields
- Encrypted notes → OKF frontmatter is inside the encrypted payload (same as now with tags)
- Export OKF button → can be removed or kept as a no-op (now redundant)
